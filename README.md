# Project : 오파고
#### 기획 내용
오목 AI게임 오픈소스를 활용하여 내부 알고리즘을 이해하고 본 강의에서 배운 내용을 접목 시켜 
프로그래밍에 대한 폭넓은 이해와 팀워크를 배움에 목적을 둔 프로젝트 입니다.
* 적용 기술
  * DB
  * GUI
  * AI
  
#### 앞으로 해야할 일
* pygame 모듈에 대한 공부[https://kkamikoon.tistory.com/129]

#### 구현하고자 하는 기능
* 시작 버튼을 통한 게임 시작
* 시작과 동시에 게임이 종료 될때 까지 시간 측정
  * 최소 시간 승리 랭킹 구현
* 졌을 경우 재도전 버튼을 통해 게임 재시작
* 이겼을 경우 승리 화면 생성
  * 승리 화면 이름 입력칸 생성
  * 이름 입력시 승리한 판에 대한 승리 시간 저장
* 저장된 기록들을 짧은 시간순으로 랭킹 표출

## Rule
##### - AI : 흑, User : 백
##### - 승리조건 : 5목 이상 (6목,, 7목 ... 등등)
##### - 3 x 3 : 흑,백 모두 금지
##### - 4 x 4 : 흑,백 모두 가능
##### - 시간제한 : 없음
## Start
##### 1. 시작버튼을 누르면 흑(AI)이 먼저 돌을 놓습니다.
##### 2. 마우스로 백(User)이 놓을 곳을 선택합니다.
##### 3. 승리조건을 만족하면 승리화면으로 넘어갑니다.
##### 4. 승리자의 이름을 입력합니다.
##### 5. 승리자의 이름을 입력을 하면 기록과 함께 순위표가 생성됩니다.
##### 6. 승리하면 재시작 버튼을 통해 재시작을 할 수 있습니다.
##### 7. 패배하면 재시작 버튼을 통해 재시작을 할 수 있습니다.

## 결론(후기)
 우리가 처음 게임이라는 장르를 선택한 이유는 많은 조원들의 관심을 가질 수 있는 공통 분야에 대한 고민을 하다<br/>
게임이라는 장르를 선택하게 되었다. 그 중에 오파고는 AI기반으로 인한 조금 색다른 게임으로 접근을 할 수 있어 흥미를
가질 수 있었다. 해당 게임이라는 장르를 접해 오픈소스를 가져오면서 추가로 구현하고자 했던 기능들은 랭킹시스템,
승패 화면 전환 기능, 버튼 추가로 매끄러운 게임 진행을 이루고자 했으며, DB를 기반으로 한 랭킹시스템을 연동을 시켜
도전 의식을 불러일으키려고 했다. 
 또한 오픈소스를 해석함에 있어 가중치라는 부분을 확인하기 위해 디버깅이나, 가중치 표출 코딩등을 시도하기도 했었다.
하지만 한정적인 프로젝트 진행 시간 및 생각보다 높은 수준의 오픈소스 코딩의 난이도로 인해 원하는 목표치에 도달하지
못하여 아쉬웠으며 프로젝트를 진행하면서 기존보다 전체적인 코딩 및 프로그래밍에 대한 이해도가 높아졌다.
 그리고 코딩에 대해 프로젝트를 하며 역할분배나 문제를 인식하여 팀활동에 대한 경험치를 쌓을 수 있었다. 다소 어려운 
문제와 익숙치 않은 프로젝트라는 과제 형식에 다소 난관이 있었으나 각자의 노력으로 잘해결해 나갈 수 있었던 것 같다.
어려운 오픈소스 코드를 해석함에 있어 다같이 머리를 맞대어 가설을 세우고 그 가설이 참으로 증명 될때의 희열과 성취
감을 느낄 수 있었다. 이렇듯 프로젝트라는 과제를 수행함에 있어 최초의 세운 목표에는 못미쳤지만 혼자라면 하지 못했
을 문제들을 함께여서 해결 할 수 있었던 것 같다.
